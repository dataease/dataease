Найти схожие местоположения

Инструмент Найти схожие местоположения вычисляет схожесть местоположений в слое поиска кандидатов с одним или более базовым местоположением на основе выбранных вами критериев. Этот инструмент может помочь ответить на вопросы, такие как

  • Какой из ваших магазинов наиболее похож на ваши наиболее успешные магазины, учитывая профили покупателей?
  • На основе характеристик населенных пунктов, более всего пострадавших от эпидемии, какие другие населенные пункты подвергаются высокому риску?
Для ответа на такие вопросы вы задаете опорные местоположения, местоположения поиска кандидатов и поля, представляющие критерии, которые вы хотите использовать. Первый входной слой должен содержать ваши опорные или реперные местоположения. Например, это может быть слой, содержащий ваши наиболее успешные магазины, или населенные пункты, сильнее всего пострадавшие от эпидемии. Затем вы указываете слой, содержащий местоположения поиска кандидатов. Это могут быть все ваши магазины или другие населенные пункты. Наконец, вы идентифицируете одно или более полей, которые будут использоваться при измерении схожести. Инструмент Найти схожие местоположения затем вычислит ранги всех местоположений поиска кандидатов по тому, насколько они совпадают с опорными местоположениями, учитывая все указанные вами поля.

Выберите слой, содержащий опорные местоположения

Слой, содержащий опорные местоположения для сопоставления.

Кроме выбора слоя из вашей карты, можно щелкнуть Выбрать слой анализа в нижней части ниспадающего списка, чтобы найти свои ресурсы для набора данных или векторного слоя файлового хранилища больших данных.

Для выборки можно использовать все местоположения

Используйте кнопки выборки для идентификации опорных местоположений, если необходимо. Опция доступна, только если объекты отрисованы на карте (не подходит для данных большого объема) Например, если входной слой содержит все местоположения – как опорные, так и местоположения поиска кандидатов – вам понадобится использовать один из инструментов выборки для идентификации опорных местоположений. Если вы создаете два отдельных слоя, один с опорными местоположениями, а второй со всеми местоположениями поиска кандидатов, вам не нужно будет делать выборку.

Если присутствует более одного опорного местоположения, отдельное местоположение создается с помощью усреднения значений каждого из полей, используемых для анализа схожести.

Поиск схожих местоположений в

Объекты поиска кандидатов в этом слое будут отсортированы - от наибольшего до наименьшего сходства.

Кроме выбора слоя из вашей карты, можно щелкнуть Выбрать слой анализа в нижней части ниспадающего списка, чтобы найти свои ресурсы для набора данных или векторного слоя файлового хранилища больших данных.

Сходство основано на

Поля, которые вы выбрали, будут использоваться в качестве критерия для вычисления сходства. Например, если вы выбрали поля численности населения и дохода, местоположения поиска кандидатов с низким (лучшим) рангом будут иметь схожие с вашими опорными местоположениями значения численности населения и дохода.

Определите сходные наименьший и наибольший с помощью

Выбранный вами метод определяет основание сопоставления.

  • Метод значений поля использует квадратные отклонения от стандартизированных значений. Используется по умолчанию.
  • Метод профилей поля использует для сравнения профиля стандартизированных значений вычисления косинусоидального подобия (коэффициента Отиаи). Для использования профилей атрибутов необходимо применение как минимум двух полей анализа.

Показать мне

Вы можете либо просмотреть все местоположения поиска кандидатов с рангами от наиболее схожих до наименее схожих, либо указать количество результатов, которые вы хотите видеть.

  • Можно задать все местоположения от наиболее до наименее схожих – все объекты в слое поиска кандидатов будут включены по порядку ранга в результирующий слой. Будет возвращено максимум 10000 объектов.
  • Если вы укажете число наиболее, наименее или наиболее и наименее похожих местоположений, вы тем самым зададите, сколько из наиболее, наименее или наиболее и наименее похожих кандидатов будут включены в выходной слой. Максимально допустимое значение - 10000.

Выберите поля для добавления результатов

Кроме того, вы можете добавить поля к вашим данным из слоя поиска. По умолчанию поля будут добавлены.

Имя слоя результата

Имя создаваемого слоя. Если вы записываете в ArcGIS Data Store, ваши результаты будут сохранены в Моих ресурсах и добавлены на карту. Если вы записываете в файловое хранилище больших данных, ваши результаты будут сохранятся в файловом хранилище больших данных и добавляться в его файл манифеста. Они не будут добавлены на карту. Имя слоя по умолчанию зависит от имени инструмента и имени входного слоя. Если слой уже существует, произойдет сбой.

Результирующий слой будет содержать опорные местоположения и указанное число отсортированных по рангу местоположений поиска кандидатов. Если имя итогового слоя уже используется, появится запрос на его переименование.

При записи в ArcGIS Data Store (реляционное или пространственно-временное хранилище больших данных) с помощью ниспадающего списка поля Сохранить результат в вы можете задать имя папки в разделе Мои ресурсы, в которую будет записан результат.
