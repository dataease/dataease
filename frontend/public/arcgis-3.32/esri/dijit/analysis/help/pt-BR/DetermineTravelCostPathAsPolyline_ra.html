Determinar Caminhos de Custo da Viagem Como Polilinha

Esta ferramenta calcula o caminho de polilinha de menor custo entre origens e destinos conhecidos.

Aplicações de exemplo incluem o seguinte:

  • Encontrar a melhor rota para uma nova estrada em termos de custos de construção.
  • Identificar o caminho a seguir a partir de vários locais suburbanos (fontes) até o shopping center mais próximo (destino).

Se Utilizar extensão de mapa atual estiver selecionada, somente estes locais nas camadas de entrada que são visíveis dentro da extensão de mapa atual serão analisados. Se não selecionada, todos os locais nas camadas de entrada serão analisadas, mesmo que estiverem fora da extensão de mapa atual.

Escolher camada de origem da feição ou raster para determinar caminhos de

Um serviço de imagem ou serviço da feição que identifica as células a partir das quais o caminho de menor custo é determinado até o destino mais barato.

Se a entrada for um serviço de imagem, a entrada consistirá em células que têm valores válidos (zero é um valor válido) e as células restantes deverão ser designadas de NoData.

Escolher camada de raster de custo da viagem

Um raster definindo o custo ou impedância para mover planimetricamente através de cada célula.

O valor de cada localização da célula representa a distância de unidade por custo para mover pela célula. Cada valor de localização de célula é multiplicado pela resolução de pixel, enquanto também compensando por movimento diagonal para obter o custo total de passagem pela célula.

Os valores do custo de raster podem ser inteiros ou pontos flutuantes, mas eles não podem ser negativos ou zero (você não pode ter um custo negativo ou zero).

Feições ou Raster de Destino de Entrada

Uma entrada que define os destinos utilizados para calcular a distância. A camada pode ser raster ou feição.

Se a entrada for um serviço de imagem, a entrada consistirá em células que têm valores válidos (zero é um valor válido) e as células restantes deverão ser designadas de NoData.

Nome da camada de polilinha resultante

O nome da camada que será criada em Meu Conteúdo e adicionada no mapa. O nome padrão é baseado no nome da ferramenta e o nome da camada de entrada. Se a camada já existir, você será solicitado para fornecer outro nome.

Você pode especificar o nome de uma pasta em Meu Conteúdo onde o resultado será salvo utilizando a caixa suspensa Salvar resultados em .

Escolher tipo de caminhos para determinar

Define a maneira pela qual os valores e zonas nos dados de destino de entrada serão interpretados nos cálculos do caminho de custo.

  • Único Melhor—Para todas as células ou locais nos dados de destino de entrada, o caminho de menor custo é derivado do local com o mínimo dos caminhos de menor custo até o local de origem. Este é o padrão
  • Para Cada Célula—Para cada célula ou local com valores válidos nos dados de destino de entrada, um caminho de menor custo é determinado e salvo no serviço da feição de saída. Com esta opção, cada célula dos dados de destino de entrada é tratada separadamente e um caminho de menor custo é determinado para cada célula.
  • Para Cada Zona—Para cada zona nos dados de destino de entrada, um caminho de menor custo é determinado e salvo no serviço da feição de saída. Com esta opção, o caminho de menor custo para cada zona começa na célula ou local com a menor ponderação de distância de custo na zona.
