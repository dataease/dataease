Agregar Pontos

Esta ferramenta funciona com uma camada de elementos de pontos e uma camada de elementos de áreas. Os elementos de área e entrada podem provir de uma camada de polígonos, ou podem ser caixas quadradas ou hexagonais calculadas aquando da execução da ferramenta. A primeira ferramenta identifica os pontos que se encontram dentro de cada área especificada. Depois de determinar esses, calcula estatísticas sobre todos os pontos da área e atribuídos à área. A estatística mais básica é a contagem do número de pontos dentro da área, mas também pode obter outras estatísticas.

Por exemplo, suponha que tem elementos de pontos com localização de lojas de café e elementos de área de condados, e pretende resumir as vendas efetuadas pelos cafés por município. Assumindo que os cafés têm o atributo TOTAL_SALES (total de vendas), pode obter a soma de todos os TOTAL_SALES em cada condado, o mínimo ou máximo de TOTAL_SALES em cada condado, ou outras estatísticas, tais como a contagem, intervalo, desvio padrão e variância.

Esta ferramenta pode também funcionar com dados com componente temporal ativa. Se a componente temporal se encontrar ativa nos pontos de entrada, as opções de divisão temporal estarão disponíveis. A divisão temporal permite-lhe calcular a relação de pontos na área, enquanto observa uma parcela temporal específica. Por exemplo, pode observar intervalos de uma hora, o que resultaria em saídas por cada hora.

Vejamos exemplo com componente temporal ativa: suponha que tem elementos de pontos por cada transação efetuada em várias localizações de cafés e não existe qualquer camada de área. Os dados foram reordenados pela duração de um ano e cada transação tem uma localização e um indicação temporal. Assumindo que cada transação tem um atributo TOTAL_SALES , pode obter a soma de todos os TOTAL_SALES no espaço e tempo que lhe sejam conveniente. Caso estas transações sejam de uma única cidade, poderíamos gerar áreas em forma de grelhas de 1 quilómetro e observar parcelas temporais semanais para resumir as transações tanto no espaço como no tempo.

Selecione uma camada que contenha pontos para agregar em áreas.

A camada de pontos que será agregada nas áreas. As camadas de pontos que são agregadas em caixas Quadrado ou Hexágono têm de ter a referência espacial de processamento definida para um sistema de coordenadas projetado que utilize os Ambientes de Análise.

Para além de escolher uma camada do seu mapa, pode escolher Escolher Camada de Análise na parte inferior da lista pendente para navegar até aos seus conteúdos de um conjunto de dados de partilha de ficheiros de big data ou camada de elementos.

Escolher uma camada de área na qual agregar

As áreas nas quais a camada de pontos será agregada. Pode escolher entre uma camada de área introduzida por si ou por caixas que são geradas enquanto a ferramenta é executada. Ao gerar caixas, por Quadrado, o número e unidades especificados determinam a altura e o comprimento do quadrado. Para Hexágono, o número e unidades especificados determinam a distância entre lados paralelos.

Análise utilizando caixas Quadrado ou Hexágono requerem um ambiente de coordenadas projetadas. Pode definir o Sistema de coordenadas de processamento nos Ambientes de Análise. Se o seu sistema de coordenadas de processamento não se encontra definido para um sistema de coordenadas projetado, será instado a defini-lo quando Correr Análise .

Selecionar a camada de àrea na qual agregar

A camada de área à qual os pontos serão agregados. Quaisquer pontos que se encontrem dentro dos limites das áreas incluídas nesta camada serão contados e, opcionalmente, resumidos, utilizando as opções de estatísticas escolhidas por si.

Quer Tamanho de Caixa para caixas, ou Camada de Área tem de ser definido.

Selecionar uma distância para gerar caixas e agregar

A distância utilizada para gerar caixas nas quais agregar os seus pontos de entrada. Para uma caixa quadrada, o tamanho representa a altura do quadrado. Para hexágono, o tamanho representa a altura do hexágono (a partir de lados paralelos).

Quer Tamanho de Caixa para caixas, ou Camada de Área tem de ser definido.

Agregar utilizando etapas temporais (opcional)

Caso a componente temporal se encontre ativa na camada de pontos de entrada, pode analisar recorrendo a etapas temporais. Existem três parâmetros que pode definir quando utiliza componente temporal:

  • Intervalo entre etapas temporais
  • Frequência de repetição da etapa temporal
  • Hora à qual alinhar as etapas temporais

Por exemplo, caso tenha dados que representam um ano no tempo e pretende analisá- los utilizando etapas semanais, defina Intervalo de etapa temporal para 1 semana.

Por exemplo, caso tenha dados que representam um ano no tempo e pretende analisá- los utilizando a primeira semana do mês, defina Intervalo de etapa temporal para 1 semana, Frequência de repetição da etapa temporal para 1 mês, e Hora à qual alinhar as etapas temporais para 1 de janeiro, às 12:00.

Intervalo entre etapas temporais

O intervalo temporal utilizado para gerar etapas temporais. Intervalo de etapa temporal pode ser utilizado isoladamente ou com o parâmetro Frequência de repetição da etapa temporal ou Hora à qual alinhar as etapas temporais ..

Por exemplo, caso pretenda criar etapas temporais que têm lugar todas as segundas-feiras entre as 9:00 e as 10:00, defina Intervalo de etapa temporal para 1 hora, Frequência de repetição da etapa temporal para 1 semana, e Hora à qual alinhar as etapas temporais para 9:00:00 à segunda.

Frequência de repetição da etapa temporal

A etapa utilizada para calcular uma etapa temporal. Frequência de repetição da etapa temporal pode ser utilizado isoladamente, com Intervalo de etapa temporal, com Hora de Referência, ou com ambos Intervalo de etapa temporal e Hora à qual alinhar as etapas temporais.

Por exemplo, caso pretenda criar etapas temporais que têm lugar todas as segundas-feiras entre as 9:00 e as 10:00, defina Intervalo de etapa temporal para 1 hora, Frequência de repetição da etapa temporal para 1 semana, e Hora à qual alinhar as etapas temporais para 9:00:00 a uma segunda-feira.

Hora à qual alinhar as etapas temporais

A data e hora utilizadas para alinhar a divisão temporal. A utilização de etapas temporais terá início às e continuará de forma regressiva desde esta hora. Caso não seja selecionada qualquer hora de referência, a etapa temporal será alinhada a 1 de janeiro de 1970.

Por exemplo, caso pretenda criar etapas temporais que têm lugar todas as segundas-feiras entre as 9:00 e as 10:00, defina Intervalo de etapa temporal para 1 hora, Frequência de repetição da etapa temporal para 1 semana, e Hora à qual alinhar as etapas temporais para 9:00:00 a uma segunda-feira.

Adicionar estatísticas (opcional)

Pode calcular estatísticas em elementos que se encontram resumidos. Pode calcular os seguintes em campos numéricos:

  • Contagem—Calcula o número de valores não-nulos. Pode ser utilizado em campos numéricos ou em strings. A contagem de [null, 0, 2] é 2.
  • Soma—A soma dos valores numéricos num campo. A soma de [null, null, 3] é 3.
  • Média—A média de valores numéricos. A média de [0, 2, null] é 1.
  • Mín—o valor mínimo de um campo numérico. O mínimo de [0, 2, null] é 0.
  • Máx—o valor máximo de um campo numérico. O valor máximo de [0, 2, null] é 2.
  • Intervalo—o intervalo de um campo numérico. Este é calculado subtraindo os valores mínimos ao valor máximo. O intervalo de [0, null, 1] é 1. O intervalo de [null, 4] é 0.
  • Variância—a variância de um campo numérico num trilho. A variância de [1] é null. A variância de [null, 1,0,1] é 0.25.
  • Desvio padrão—O desvio padrão de um campo numérico. O desvio padrão de [1] é null. O desvio padrão de [null, 1,0,1,1] é 0.5.

Pode calcular os seguintes em campos de string:

  • Contagem—O número de strings não-nulas.
  • Qualquer—Esta estatística é uma amostra aleatória de um valor de string no campo especificado.
Todas as estatísticas são calculadas em campos que não são nulos. A camada resultante irá conter um novo campo por cada estatística calculada. É possível adicionar qualquer número de estatísticas, selecionando um atributo e uma estatística.

SpatialReference (wkid)

Este é um parâmetro temporário de pré-lançamento para definir a referência espacial. Muitas ferramentas de big data requerem que o sistema de coordenadas projetado seja utilizado como referência espacial para processamento. Por predefinição, a ferramenta utilizará o sistema de coordenadas de entrada, mas irá falhar no caso de o mesmo se tratar de um sistema de coordenadas geográficas. Para definir um sistema de coordenadas projetado, introduza o WKID. Por exemplo, Web Mercator seria introduzido como 3857.

Selecionar um ArcGIS Data Store no qual guardar os resultados

Os resultados de GeoAnalytics são guardados num armazenamento de dados e expostos como uma camada de elementos em Portal for ArcGIS. Na maioria dos casos, os resultados deverão ser armazenados num armazenamento de dados espaciotemporal, sendo este o comportamento predefinido. Em alguns casos, será uma boa opção guardar os resultados no armazenamento de dados relacional. As seguintes são razões que justificam o armazenamento de resultados no armazenamento de dados relacional.

  • Pode utilizar resultados em colaboração portal-to-portal.
  • Pode ativar funcionalidades de sincronização com os seus resultados

Não deverá utilizar o armazenamento de dados relacional caso antecipe um aumento dos seus resultados de GeoAnalytics e necessite de tirar partido das funcionalidades de armazenamento de big data espaciotemporais para gerir grandes quantidades de dados.

Nome da camada resultante

O nome da camada que será criada. Se estiver a escrever para ArcGIS Data Store, os seus resultados serão gravados em O Meu Conteúdo e adicionados ao mapa. Se estiver a escrever para uma partilha de ficheiros big data, os seus resultados serão guardados numa partilha de ficheiros big data e adicionados ao seu manifesto. Não serão adicionados ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Caso a camada já existe, a ferramenta irá falhar.

Ao escrever para ArcGIS Data Store (armazenamento relacional ou espaciotemporal de dados big data) utilizando a caixa suspensa Guardar resultado em pode especificar o nome de uma pasta em O Meu Conteúdo, onde será armazenado o resultado.
