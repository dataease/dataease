Ligar Elementos

Ligar Elementos funciona com duas camadas e liga atributos entre um elemento e outro com base em relações espaciais e de atributos. A ferramenta determina todos os elementos de entrada que correspondem às condições de ligação definidas e anexa a camada de ligação à camada alvo.

Ligar Elementos pode ser aplicado a pontos, linhas, áreas e tabelas. Uma ligação espacial requer que os seus dados possuam uma geometria.

Caso Utilizar a atual extensão de mapa se encontre selecionado, apenas serão analisados os elementos que se encontrem visíveis na atual extensão do mapa. Se não estiver selecionado, todos os elementos da camada alvo e da camada de ligação serão analisados, mesmo que se encontrem fora da atual extensão de mapa.

Selecionar camada alvo

A camada alvo que terá atributos da camada de ligação anexada à respetiva tabela.

Além de escolher uma camada do seu mapa, pode selecionar Escolher Camada de Análise que se encontra na parte inferior da lista pendente. Esta opção abre uma galeria, na qual pode navegar e selecionar conjuntos de dados de várias fontes para utilizar na sua análise.

Selecionar camada para ligar à camada alvo

A camada de ligação com os atributos que serão anexados à camada alvo.

Além de escolher uma camada do seu mapa, pode selecionar Escolher Camada de Análise que se encontra na parte inferior da lista pendente. Esta opção abre uma galeria, na qual pode navegar e selecionar conjuntos de dados de várias fontes para utilizar na sua análise.

Selecionar o(s) tipo(s) de ligação

Pode aplicar uma ligação espacial, uma ligação de atributos ou uma combinação de ambas.

  • Espacial—Recorre a uma relação espacial especificada para ligar elementos. Isto exige que ambas as camadas possuam geometria.
  • Atributo—Liga elementos com base em campos iguais.

Escolher uma relação espacial

A relação espacial que determinará se os elementos se ligam uns aos outros. As relações disponíveis dependerão do tipo de geometria (ponto, linha, áreas) utilizada como elementos de entrada. As relações disponíveis incluem as seguintes:

  • Idênticos a—Os elementos da camada alvo terão correspondência caso sejam idênticos a elementos da camada de ligação.
  • Interseta—Os elementos da camada alvo terão correspondência caso intersetem elementos da camada de ligação.
  • Contém completamente—Os elementos terão correspondência caso os elementos da camada alvo contenham completamente elementos da camada de ligação.
  • Completamente contidos—Os elementos terão correspondência caso os elementos da camada alvo se encontrem completamente contidos em elementos da camada de ligação.
  • A uma distância de—Os elementos da camada alvo terão correspondência caso se encontrem a uma distância especificada de elementos da camada de ligação.

Ligação de atributos

A relação de atributos que determinará se os elementos se ligam uns aos outros. Os elementos são correspondidos quando os valores de campo na camada de ligação são iguais a valores de campo da camada alvo.

Por exemplo, suponha que tem uma camada geográfica com os endereços residenciais de todo um condado (incluindo um campo com o código postal ZIP) e uma tabela com dados demográficos de saúde por código postal (incluindo um campo chamado HEALTHZIP). Pode ligar os dados de saúde aos dados residenciais ao corresponder o campo ZIP ao campo HEALTHZIP, o que resultará numa camada de residências com os seus correspondentes dados de saúde.

Escolher operação de ligação

Determina como as ligações entre a camada alvo e a camada de ligação serão geridas caso se determine que vários elementos na camada de ligação têm a mesma relação com a camada alvo. As seguintes são duas operações de ligação de entre as quais escolher:

  • Ligar one to one—Esta opção liga os elementos da camada alvo a um único elemento na camada de ligação. O modo como os elementos são ligados é determinado pelo parâmetro Adicionar Estatísticas ou o parâmetro Definir que registo é mantido . Se Adicionar estatísticas for utilizado, todos os elementos ligados correspondidos serão resumidos para cada elemento na camada alvo. A contagem de elementos ligados será adicionada por predefinição.Se Definir que registo é mantido for utilizado, pode optar entre manter o primeiro registo correspondente (predefinição) ou manter um registo correspondente baseado numa expressão. Uma expressão pode utilizar um campo numérico (opção para registo de maior ou de menor dimensão) ou um campo de data (opção para mais recente ou mais antigo).
  • Ligar one to many—Esta opção liga todos os elementos da camada de ligação correspondentes à camada de alvo. A camada resultante conterá múltiplos registos do elemento alvo.

Por exemplo, suponha que pretende encontrar supermercados que se encontrem até 5 quilómetros de um mercado agrícola. Neste caso, a camada a ligar tem um único elemento que representa um mercado agrícola, e os elementos em ligação representam as mercearias locais que possuem incluindo atributos, como o total de vendas anuais. Utilizando a ferramenta Ligar Elementos , descobrirá que cinco mercearias correspondem a esses critérios. Caso especifique uma operação de ligação Ligar one to many, no final terá cinco elementos no resultado, com cada linha a representar o mercado agrícola e um supermercado. Caso especifique uma relação Ligar one to one e selecione uma estatística, no final terá um elemento representando o mercado agrícola e as informações resumidas referentes aos supermercados, tais como a contagem (5), e outras estatísticas opcionais, como a soma de vendas anuais.

Adicionar estatísticas (opcional)

Caso se encontre selecionada uma opção Ligar one to one para campos numéricos, pode resumir e calcular estatísticas em elementos ligados. Por predefinição, a contagem de elementos ligados com correspondência será calculada e adicionada à camada resultante. Caso opte por não acrescentar estatísticas adicionais, o primeiro elemento com correspondência da camada de ligação será ligado ao primeiro elemento com correspondência da camada alvo. Pode calcular uma ou mais das opções que se seguem:

  • Soma
  • Mínimo
  • Máximo
  • Média
  • Desvio Padrão

Nome da camada resultante

Este é o nome da camada que será criada em O Meu Conteúdo e adicionada ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Se o nome da camada já existir, ser-lhe-á pedido que introduza outro nome.

Ao utilizar o menu suspenso Guardar resultado em pode definir o nome da pasta em O Meu Conteúdo onde o resultado será guardado.

Criar resultados como vista de camada de elementos alojada

Criar resultados como uma vista da camada de elementos alojada permitirá que os dados permaneçam atualizados, enquanto os dados da origem se alteram. Camadas de visualização alojadas de elementos que contêm ligações (joins) serão apenas de leitura e não consomem créditos de análise ou armazenamento.

Pode criar uma como de camada de elementos alojada a partir dos resultados de análise caso todas as condições que se seguem se verifiquem:

  • É proprietário de ambas as camadas.
  • Ambas as camadas são camadas de elementos alojadas.
  • Está a efetuar uma ligação baseada em atributos.
