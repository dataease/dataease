ポータルでの解析の環境設定

Portal for ArcGIS および ArcGIS Online での解析の環境設定。

結果レイヤーを上書き

既存のレイヤーを上書きするかどうかを指定します。オンにすると、同じ名前のレイヤーがコンテンツ内にすでに存在する場合は、そのレイヤーが上書きされます。

  • オン - 同じ名前のレイヤーがコンテンツ内にすでに存在する場合は、そのレイヤーが上書きされます。出力の座標系が入力と同じになります。
  • オフ - 指定された結果レイヤーの名前がすでに存在する場合は、エラーが発生します。ツールを実行する前に結果レイヤーの名前を変更しておく必要があります。これがデフォルトです。

分析の実行後に分析ウィジェットを閉じる

ツールの実行中に解析ダイアログを開いたまま有効な状態にしておくかどうかを指定します。この設定を使用すると、前の解析ジョブが継続中の場合でも、解析パラメーターを変更することができます。解析は反復的なプロセスであるため、ツール ダイアログ ボックスを開いたままにしておくことで、パラメーターを変更し、解析を再実行できます。

  • オン - 解析ツール ダイアログ ボックスが閉じて、[コンテンツ] ウィンドウにフォーカスが戻されます。これがデフォルトです。
  • オフ - 解析ツール ダイアログ ボックスが開いたまま有効な状態になります。

解析結果をホスト フィーチャ レイヤーとして保存

解析から作成する結果レイヤーをフィーチャ コレクションにするかホスト フィーチャ レイヤーにするかを指定します。

  • オン - 解析結果がホスト フィーチャ レイヤーとして作成され、アイテムがコンテンツ内に作成されます。これがデフォルトです。
  • オフ - 解析結果がフィーチャ コレクションとして作成されます。

出力座標系

ラスター解析の結果レイヤーの座標系を指定します。

  • 入力データと同様 - 解析結果の座標系が入力と同じになります。これがデフォルトです。
  • 値指定 - 解析結果の座標系は選択した座標系になります。このオプションをオンにした場合は、地球ボタンをクリックして既知の座標系のリストから座標系を選択するか、指定のボックスに空間参照の WKID を追加します。
  • レイヤー <名前> - 解析結果の座標系は、Web マップで選択した既存のレイヤーと同じになります。

処理に使用する座標系

GeoAnalytics 解析の実行中に使用される座標系を指定します。

GeoAnalytics での一部の解析では、処理を投影座標系で実行する必要があります。データまたは距離単位のラスター化を実装しているツールでは投影座標系が必要です。

  • 入力データと同様 - 解析が入力と同じ座標系で処理されます。これがデフォルトです。
  • 値指定 - 解析の座標系は選択した座標系になります。このオプションをオンにした場合は、地球ボタンをクリックして既知の投影座標系のリストから投影座標系を選択するか、指定のボックスに空間参照の WKID を追加します。
  • レイヤー <名前> - 解析の座標系は、Web マップで選択した既存のレイヤーと同じになります。

範囲

解析の実行中に使用される範囲または境界を指定します。指定の範囲内に完全に含まれているか、指定の範囲と交差しているすべての入力フィーチャが解析に使用されます。

  • デフォルト - このツールから提供される範囲。
  • 値指定 - 範囲は、指定した座標系によって定義されます。
  • レイヤー <名前> - 解析の処理に使用される範囲は、Web マップで選択した既存のレイヤーの空間範囲と同じになります。

スナップ対象ラスター

ラスター解析で指定した [スナップ対象ラスター] レイヤーのセルの配置に一致するように、出力ラスター レイヤーの範囲を調整します。

セル サイズ

ラスター解析で出力ラスター レイヤーの作成に使用されるセル サイズまたは解像度を指定します。デフォルトの出力解像度は、入力ラスター レイヤーの最も大きいセル サイズによって決まります。

  • 入力データの最小セルサイズ - すべての入力レイヤーの中から最小のセル サイズを使用します。
  • 入力データの最大セルサイズ - すべての入力レイヤーの中から最大のセル サイズを使用します。これがデフォルトです。
  • 値指定 - 数値を指定してセル サイズを設定します。このオプションを選択すると、デフォルト値が「1」に設定されます。
  • レイヤー <名前> - セル サイズを選択されたラスター レイヤーに設定します。

マスク

ラスター解析で対象エリアの定義に使用されるレイヤーを指定します。解析マスク内にあるセルだけが解析処理で考慮されます。

  • マスクは、ラスターでもフィーチャ レイヤーでもかまいません。
  • 解析マスクがラスターの場合、値を持つすべてのセルがそのマスクを指定すると見なされます。マスク ラスター内の NoData のセルはマスクの外にあると見なされ、解析結果レイヤーでも NoData になります。
  • 解析マスクがフィーチャ レイヤーの場合は、実行時に内部でラスターに変換されます。そのため、解析に適した [セル サイズ] と [スナップ対象ラスター] が設定されていることを確認します。

データ ストア

GeoAnalytics の結果は、 ArcGIS Data Store に保存し、フィーチャ レイヤーとして Portal for ArcGIS に公開するか、構成済みのビッグ データ ファイル共有に保存できます。

結果を ArcGIS Data Store に保存する場合、一般的に、結果はビッグ データ ストアに保存されます。これがデフォルトです。結果をリレーショナル データ ストアに保存する理由は以下のとおりです。

  • 結果をポータル間での連携に使用する場合
  • 結果との同期機能を有効にする場合

GeoAnalytics の結果が増加すると予想されており、大量のデータを処理するビッグ データ ストアの機能を利用したい場合には、リレーショナル データ ストアを使用すべきではありません。

並行処理ファクター

複数のラスター処理サービス インスタンス間で解析を分散処理します。

[並列処理ファクター] 環境に対応しているツールを使用すると、データの処理に使用できるラスター処理サービス インスタンスの数を制御することができます。

ツールが [プロセッサー タイプ] に対応していないか、 [プロセッサー タイプ] 環境が [CPU] に設定されている場合は、 [並列処理ファクター] 環境によってラスター処理 (CPU) サービス インスタンスを制御します。 [プロセッサー タイプ] が [GPU] に設定されている場合は、 [並列処理ファクター] 環境によってラスター処理 GPU インスタンスの数を制御します。

[並列処理ファクター] の設定により、ラスター解析イメージ サーバーで 1 つのラスター解析タスクの処理に使用される並列ワーカーの数を要求できます。ただし、並列処理の総数がラスター処理 (CPU または GPU) サービス インスタンスの最大数を超えると、追加の並列処理がキューに配置されます。

[並列処理ファクター] が指定されていない (これがデフォルトです) 場合、ツールでは、ラスター処理サービス インスタンスの最大数の 80 パーセントが使用されます。整数またはパーセント値のいずれかを並列処理ファクターとして指定できます。

プロセッサー タイプ

[プロセッサー タイプ] 環境に対応しているツールを使用すると、データを処理する場所と方法を選択できます。サーバー コンピューターの CPU または GPU のいずれかを使用してデータを処理できます。 [プロセッサー タイプ] 環境が指定されない場合、ツールでは、データの処理に CPU が使用されます。これがデフォルトです。

CPU 処理は、 [並列処理ファクター] による処理と同様に、複数のコアおよびインスタンス間で並列化できます。

GPU はグラフィックスと画像の処理に有効であり、高度な並列構造をしているため、繰り返し方式で大量のデータを効率よく処理できます。この環境に対応しているラスター解析ツールは、 [並列処理ファクター] による処理と同様に、複数のラスター解析サーバー コンピューターの GPU インスタンス間でそのジョブを分散処理できます。

  • GPU - データはグラフィックス処理装置 (GPU) を使用して処理されます。
  • CPU - データはコンピューターの中央処理装置 (CPU) を使用して処理されます。
