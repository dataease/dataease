Paramètres de l'environnement d'analyse du portail

Paramètres d’environnement pour l’analyse dans Portal for ArcGIS et ArcGIS Online.

Remplacer la couche de résultat

Spécifie s’il faut remplacer les couches existantes. Lorsque cette option est sélectionnée, s'il existe une couche du même nom dans votre contenu, elle sera remplacée.

  • Cochée : s'il existe une couche du même nom dans votre contenu, elle sera remplacée. La sortie présentera le même système de coordonnées que les données en entrée.
  • Non cochée : si le nom de la couche de résultat existe déjà, une erreur se produit et la couche de résultat doit être renommée pour que l’outil puisse être exécuté. Il s’agit de l’option par défaut.

Fermer le widget d'analyse après l'exécution de l'analyse

Spécifie si la boîte de dialogue d’analyse reste ouverte et activée pendant l’exécution de l’outil. Ce paramètre permet de modifier vos paramètres d’analyse alors que l’analyse précédente se poursuit. L’analyse est un processus itératif et en gardant la boîte de dialogue de l’outil ouverte, vous pouvez modifier les paramètres et exécuter une nouvelle fois l’analyse.

  • Cochée : la boîte de dialogue de l’outil d’analyse se ferme et vous êtes redirigé vers la table des matières. Il s'agit du paramètre par défaut.
  • Non cochée : la boîte de dialogue de l’outil d’analyse reste ouverte et activée.

Stocker les résultats de l'analyse en tant que couches d'entités hébergées

Indique si la couche de résultats créée à partir de votre analyse sera une couche d’entités hébergées ou une collecte d’entités.

  • Cochée : les résultats de votre analyse généreront une couche d’entités hébergées, et un élément sera créé dans votre contenu. Il s’agit de l’option par défaut.
  • Non cochée : les résultats de votre analyse vont générer une collecte d'entités.

Système de coordonnées en sortie

Spécifie le système de coordonnées de la couche de résultats pour l’analyse raster.

  • Identique à l'entrée : le résultat de votre analyse apparaîtra dans le même système de coordonnées que les données en entrée. Il s’agit de l’option par défaut.
  • As specified (Comme spécifié) : les résultats de votre analyse apparaîtront dans le système de coordonnées que vous choisissez. Lorsque cette option est sélectionnée, cliquez sur le bouton représentant le globe et choisissez un élément dans la liste des systèmes de coordonnées connus ou ajoutez le WKID de référence spatiale dans la zone appropriée.
  • Layer <name> (Couche <nom>) : les résultats de votre analyse apparaîtront dans le même système de coordonnées que toute couche existante choisie dans votre carte web.

Traitement du système de coordonnées

Spécifie le système de coordonnées que l’analyse utilisera pendant l’exécution de la géoanalyse.

Pour la géoanalyse, certaines analyses exigent que le traitement soit exécuté dans un système de coordonnées projetées. Tous les outils ayant classifié des données ou des unités linéaires nécessite un système de coordonnées projetées.

  • Identique à l'entrée : votre analyse sera traitée dans le même système de coordonnées que les données en entrée. Il s’agit de l’option par défaut.
  • As specified (Comme spécifié) : votre analyse apparaîtra dans le système de coordonnées que vous choisissez. Lorsque cette option est sélectionnée, cliquez sur le bouton représentant le globe et choisissez un élément dans la liste des systèmes de coordonnées projetées ou ajoutez le WKID de référence spatiale dans la zone appropriée.
  • Layer <name> (Couche <nom>) : votre analyse apparaîtra dans le même système de coordonnées que toute couche existante choisie dans votre carte web.

Etendue

Indique l’étendue ou la limite utilisée pendant l’exécution de l’analyse. Toutes les entités en entrée se trouvant intégralement dans l'étendue spécifiée ou qui intersectent cette dernière seront utilisées dans l'analyse.

  • Default (Par défaut) : étendue fournie par l’outil.
  • As specified (Comme spécifié) : l’étendue est définie par les coordonnées que vous avez fournies.
  • Layer <name> (Couche <nom>) : l’étendue utilisée pour traiter votre analyse sera identique à l’étendue spatiale d’une couche existante que vous avez choisie dans votre carte web.

Raster de capture

Ajuste l’étendue de la couche raster en sortie afin qu’elle corresponde à l’alignement des cellules de la couche Snap Raster spécifiée dans l’analyse raster.

Taille de cellule

Spécifie la taille de cellule ou la résolution qui sera utilisée pour créer la couche raster en sortie dans l’analyse raster. La résolution en sortie par défaut est déterminée par la taille de cellule la plus importante dans la couche raster en entrée.

  • Minimum d’entrées : utilise la plus petite taille de cellule de toutes les couches en entrée.
  • Maximum d’entrées : utilise la plus grande taille de cellule de toutes les couches en entrée. Il s’agit de l’option par défaut.
  • Comme spécifié : indique une valeur numérique pour définir la taille de cellule. Si cette option est sélectionnée, la valeur par défaut est 1.
  • <Nom> de la couche : définit la taille de cellule sur la couche raster choisie.

Masque

Spécifie une couche qui permettra de définir votre zone d’intérêt dans l’analyse raster. Seules les cellules comprises dans le masque d'analyse seront prises en compte lors de l'analyse.

  • Le masque peut être un raster ou une couche d'entités.
  • Si le masque d'analyse est un raster, toutes les cellules ayant une valeur sont prises en compte pour définir le masque. Les cellules d'un raster de masque de type NoData seront considérées comme étant en dehors du masque et auront donc une valeur NoData dans la couche de résultat d'analyse.
  • Si le masque d'analyse est une couche d'entités, il est converti en interne en raster lors de l'exécution. Vous devez donc vous assurer que les options Cell Size (Taille de cellule) et Snap Raster (Raster de capture) sont correctement paramétrés pour votre analyse.

Magasin de données

Les résultats de la géoanalyse sont stockés dans une instance ArcGIS Data Store et s’affichent en tant que couche d’entités dans Portal for ArcGIS ou dans un partage de fichiers Big Data configuré.

Lorsque vous conservez un résultat dans ArcGIS Data Store, les résultats doivent figurer, dans la plupart des cas, dans le répertoire de données spatio-temporelles. Il s'agit de l'option par défaut. Voici les raisons pour lesquelles stocker les résultats dans un stockage des données relationnelles :

  • Utiliser vos résultats dans une collaboration de portail à portail
  • Permettre la synchronisation de vos résultats

Vous ne devez pas utiliser un stockage des données relationnelles si vous pensez que vos résultats GeoAnalytics vont augmenter et que vous souhaitez tirer part des fonctionnalités du stockage de Big Data spatio-temporelles pour gérer les grands volumes de données.

Facteur de traitement parallèle

Répartit l’analyse sur plusieurs instances du service de traitement raster.

Les outils qui respectent l’environnement Parallel processing factor (Facteur de traitement parallèle) permettent de contrôler le nombre d’instances du service de traitement raster pouvant être utilisés en vue du traitement des données.

Si l’outil ne respecte pas le type de processeur (Processor type) ou si l’environnement Processor type (Type de processeur) est défini sur CPU, l’environnement Parallel processing factor (Facteur de traitement parallèle) contrôle les instances de service (CPU) de traitement raster. Si Processor type (Type de processeur) est défini sur GPU, l’environnement Parallel processing factor (Facteur de traitement parallèle) contrôle le nombre d’instances GPU de traitement raster.

En configurant l’environnement Parallel processing factor (Facteur de traitement parallèle), vous pouvez demander le nombre de processus parallèles que le serveur d’imagerie d’analyse raster utilise pour traiter une tâche d’analyse raster. En revanche, si le nombre total de processus parallèles dépasse le nombre maximum d’instances de service (CPU or GPU) de traitement raster, les processus parallèles supplémentaires seront mis en attente.

Si le facteur de traitement parallèle n’est pas spécifié (option par défaut), l’outil utilisera 80 pour cent du nombre maximum d’instances de service de traitement parallèle. Un nombre entier ou un pourcentage peut être spécifié en tant que facteur de traitement parallèle.

Type de processeur

Les outils qui respectent l’environnement Processor type (Type de processeur) permet de choisir l’emplacement et la manière de traiter vos données. Vous pouvez traiter vos données à l’aide de l’unité de traitement centrale de l’ordinateur (CPU) ou de l’unité de traitement graphique (GPU) du serveur. Si l’environnement Processor type (Type de processeur) est vide, l’outil utilise le CPU pour traiter les données. Il s’agit de l’option par défaut.

Le traitement CPU peut être parallélisé sur plusieurs cœurs et instances, comme avec l’environnement Parallel processing factor (Facteur de traitement parallèle).

Les GPU sont efficaces pour traiter les graphiques et les images. Leur structure hautement parallèles leur permet de traiter efficacement des blocs de données volumineux de manière répétitive. Les outils d’analyse raster qui respectent cet environnement peuvent distribuer leur travail sur plusieurs instances GPU au niveau de plusieurs serveurs d’analyse raster, comme avec l’environnement Parallel processing factor (Facteur de traitement parallèle).

  • GPU (Unité de traitement graphique) : les données seront traitées à l’aide de l’unité de traitement graphique.
  • CPU (Unité de traitement centrale) : les données seront traitées à l’aide de l’unité de traitement centrale de l’ordinateur.
